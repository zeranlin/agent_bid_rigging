# 长 PDF 格式支持计划

## 目标

支持以单个长 PDF 形式交付、并具有章节结构的招投标材料。

## 范围

纳入范围：

- 章节感知的 PDF 解析
- 关键表格抽取
- 章节级事实融合
- 面向章节的评分输入
- 报告证据升级为章节/页码/片段引用

首轮不纳入范围：

- 全量 OCR-first 文档理解
- 纯扫描件语料支持
- 高级版面/视觉基准测试

## 交付物

### 第一阶段：章节解析

- `capabilities/pdf_sectioning`
- 每个 PDF 的章节列表
- 章节标题、起始页、结束页、章节文本

### 第二阶段：表格抽取

- 开标表抽取
- 报价表抽取
- 偏离表抽取
- 业绩表抽取

### 第三阶段：事实融合

- `ReviewFacts` 扩展为支持章节感知与表格感知事实
- 来源元数据中包含章节与页码

### 第四阶段：评分

- 面向长 PDF 格式更新结构相似性
- 技术/实施/偏离等章节比较规则
- 对通用承诺/培训类文本继续降权

### 第五阶段：报告

- `formal_report.md` 引用章节/页码/片段
- 附表行引用章节/页码/片段
- `opinion.md` 使用同一套章节感知证据摘要

## 实施步骤

1. 在 `capabilities` 下增加 PDF 章节解析器
2. 产出按文档划分的章节目录
3. 为开标/报价/偏离/业绩章节增加表格抽取辅助
4. 扩展 fusion，把章节与表格输出映射进 `ReviewFacts`
5. 更新 scoring 以消费新事实
6. 更新报告生成器，引用章节/页码/片段
7. 为新 PDF 样本集补充测试

## 验收标准

- 新招标 PDF 能解析成可用章节
- 每份投标 PDF 至少能产出：
  - 投标函章节
  - 资格相关章节
  - 存在时的报价/开标章节
  - 存在时的技术/实施/培训章节
- 报价金额能从开标/报价章节中抽出
- 长文档章节相似性可在不把 OCR 作为主路径的前提下完成分析
- 报告保持可读，证据引用可追溯

## 风险

- 投标 PDF 可能使用不一致的标题样式
- 部分表格在纯文本抽取中会严重扁平化
- 某些“技术相似”属于合理模板复用，需要降权
- 扫描资格页可能仍需 OCR 兜底

## 说明

本计划复用现有总体架构。
它不会替换 `ReviewFacts`、`strategy`、`scoring` 或 `llm_review`。
它做的是新增面向长 PDF 格式的能力层，并通过更结构化的事实接回现有审查主链。
